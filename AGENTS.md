## Architecture — what is non-obvious

### Convex-first policy

Read `docs/CONVEX-FIRST-ENGINEERING-STANDARD.md` before backend work if present. Key rules:
- Prefer Convex queries/mutations/actions over external orchestration.
- Schema changes are explicit; every write must satisfy required fields.
- Use Convex generated API imports (`convex/_generated/api`), not custom aliases.
- Mutations must be idempotent, especially tracking/streak logic.
- AI paths should fail-soft with deterministic fallbacks where product-safe.

### Two Convex directories

- `convex/` — backend functions deployed to Convex: schema, auth, mutations, actions, HTTP routes, crons.
- `src/convex/` — client-side Convex helpers used by the frontend.

Do not confuse them. Backend logic goes in `convex/`; client wrappers go in `src/convex/`.

### Video pipeline — V2 only

- Entry: `src/lib/video/pipeline/index.ts`.
- The old V1/V2 feature-flag fallback is retired. `getPipeline()` returns `V2Pipeline`.
- V2 lives under `src/lib/video/v2/` and uses the stage flow: Planner → KeyframeGenerator → SceneAnimator → Assembler.
- Do not reintroduce V1 as the production/default path.
- V1 code may remain temporarily for parity/reference tests, but active local, worker, GitHub/Vercel, and production paths should exercise V2.

### Modal FLUX required for video scene images

- Required generator: `src/lib/video/generators/modal-flux.ts`.
- Modal endpoint: `modal/flux-image.py`.
- The TypeScript Modal file `modal/flux-image.ts` is deprecated historical reference.
- `MODAL_FLUX_ENDPOINT_URL` must be configured for real image generation.
- Do not reintroduce OpenAI image generation, DALL-E, Qwen fallback, or silent local image fallback for video scene images.
- OpenAI may still be used for unrelated text/clarity/TTS paths, but not video scene images.

### Render flow

Next.js `/api/render` route → checks auth + movie state → either:

1. **Local**: FFmpeg pipeline runs in-process when `RENDER_WORKER_URL` is omitted.
2. **Remote**: Posts job to render worker → worker runs FFmpeg → uploads MP4 to R2 → POSTs webhook to Convex (`/render-complete` or `/render-fail`).

Render jobs use `renderJobId` for idempotency. The Convex HTTP router handles webhook callbacks.

### Auth

Runtime auth direction is Convex native auth (`@convex-dev/auth`) with Google/GitHub providers. Treat old Clerk references as stale unless current code proves otherwise.

## Key env vars by location

Never print actual secret values.

- `NEXT_PUBLIC_CONVEX_URL`
- `NEXT_PUBLIC_APP_URL`
- `MODAL_FLUX_ENDPOINT_URL`
- `MODAL_FLUX_API_KEY` or `MODAL_TOKEN_SECRET` if the app calls Modal directly

## Gotchas

- `convex/crons.ts` is a disabled placeholder unless later work wires safe internal scan functions.
- `convex/clarityAgent.ts` may be a stub; verify before relying on it.
- `src/lib/mindmovie/` and `src/lib/mindMovies/` are separate directories and both may be in use.
- Path alias `@/*` maps to `./src/*`.
- Video rendering needs FFmpeg and Python available locally/worker-side.
- R2 upload success does not guarantee browser playback; verify CORS/public-read/CSP/media-src on the watch page.
